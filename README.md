# Resource-Management-System (26T0019 Kanna Nakagome)
組織内のプロジェクトと社員のリソース（工数）を適切に管理するためのシステムを作成しました。
ユーザーは付与されたロール（権限）に応じて、以下の機能を利用できます。

## ロール（権限）

* **Admin（管理者）**: システム全体の管理・リソース配分を行う権限
* **Employee（一般社員）**: 自身の稼働枠と工数予定を管理する権限
* ※ Superuser（特権管理者）は、アカウント作成時点で自動的に Admin ロールが付与されます。

---

## 認証・アカウント管理

* **ログイン機能**: 登録済みのアカウントでシステムにログインします。
* **新規ユーザー登録機能**: 利用者が自身でアカウントを作成できます。
  * 新規登録画面から作成されたアカウントには、デフォルトで **Employee（一般社員）** ロールが自動付与されます。
  * ※ 管理者が「社員一覧」から新規追加を行う場合に限り、任意のロール（Admin等）を設定して登録することが可能です。
* **アカウント情報編集（全ユーザー共通）**: Admin・Employee問わず、全ユーザーが自身の「氏名」の編集、および「メールアドレス」の登録・更新を行うことができます。（2項目だと少ないと感じたのでメールアドレス欄を追加しただけで機能への影響はないです）

---

## Admin（管理者）の機能

管理者は、組織全体のリソース最適化とプロジェクト管理を行います。（※ Employeeの全機能も利用可能です）

### 社員管理
* 登録されている全社員の一覧閲覧
* 各社員のアカウント権限（ロール）の変更

### プロジェクト管理
* プロジェクト一覧の閲覧
* プロジェクトの新規登録および編集
* ※ プロジェクト登録時は「プロジェクト名」と「必要工数」の設定が必須となります。

### リソース管理
* 各社員のリソース稼働状況の確認（月次工数、割当済み工数、残工数、アサイン中プロジェクト一覧）
* 社員ごとの「月次稼働枠」の編集（代理設定）
* 社員に対する「プロジェクト工数」の新規割り当て
* すでに割り当てられている工数の変更・調整

---

## Employee（社員）の機能

一般社員は、自身の毎月の稼働状況とアサイン状況をシステムに登録・報告します。

### 稼働枠管理
* 自身の月毎の稼働工数（最大稼働枠）の登録および更新

### プロジェクト別工数管理
* 自身に割り当てられたプロジェクトに対する、工数の登録および更新
